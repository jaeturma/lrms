# Roadmap

Reconciled against `.ai/07-modules.md` (generated by direct code inspection of routes, controllers, models, migrations, and tests). See that file for per-module completion %, feature lists, and gaps. This file gives the coarse, roadmap-level view; treat `07-modules.md` as the source of truth when the two disagree.

## Completed

✅ Authentication & Account Security (login, admin login, OTP password reset, 2FA)

✅ Admin & School Dashboards (executive KPI tiles + charts, scope filters)

✅ School Directory, Self-Activation Onboarding & Bulk Import

✅ Location / School Year / Grade Level / Resource Type Reference Data

✅ Printed Learning Resources (title catalog, school encoding, inventory ledger)

✅ Resource Distribution (division → school delivery + receipt)

✅ Digital Learning Materials Repository (catalog + school browsing + import)

✅ ICT / SME / Other Equipment (catalog + school inventory + movement log)

✅ School Enrollment

✅ Application Settings (branding & SMTP) and Content Management (Support/About)

## In Progress / Needs Enhancement

🟡 Division Reports — functional (resource adequacy, equipment breakdowns, 4 CSV exports) but still tabular-only; no charts (unlike the dashboards), no PDF output, no scheduling

🟡 Equipment lifecycle extras — condition/status tracking and movement history work, but there's no maintenance/warranty-expiry alerting, depreciation reporting, or a formal disposal/condemnation workflow

## Pending

⬜ Analytics — dashboards already cover KPI tiles and trend charts; still missing drill-down navigation (click a chart segment → filtered list), saved/bookmarked views, and predictive or year-over-year comparative analytics

⬜ QR Labels — `barcode`/`qr_code` exist as plain text columns on all three equipment tables today; no label image generation, printing, or scan-to-lookup exists yet

⬜ AI Recommendations — not started

⬜ Regional Reports — no region/multi-division data model exists in the schema at all; every report and dashboard is scoped to a single division. This is an architecture gap, not just a missing report

⬜ Object Storage — file uploads (logos, resource covers/attachments, digital LM files) go to the local `public` disk (`Storage::disk('public')`); no cloud object storage (S3-compatible) is wired up
